上面这张图是代价函数的图。我们这里先引入“代价函数”这个概念，接下来我会慢慢一步一步说。

函数图上的点，是一些数据。我们现在举个例子。假设现在我们要计算机去预测一个房子的价格，x代表房子的平方数，y代表房子的价格。现在我们把这些数据给了计算机，计算机把这些数据分布到图上，形成了上面的那些点。

我们上面说了，监督学习是给计算机一些我们已知解答的数据来求输入输出的关系。也就是说，我们希望给了这些数据之后，我们再给计算机一个房子的平方数x，计算机可以给出它预测的一个房价的价格y。

但问题来了，房子的价格和房子的平方数有关系，但不一定同样大小的房子就是同样的价格，可能还有其它因素影响着房价（例如附近是否交通发达）。计算机是怎么得出结论呢 ？

这里很明显，计算机只能通过已有的数据计算出一个与现实相近的解答。而这，就是数学的线性回归问题。线性回归的定义各位同学可以百度以下。不懂线性回归的同学可以理解为：计算机要总结出一个公式。这个公式在图上画一条线，每个点离这条线的距离的总和h，是所有情况中最小的。

图中的那条蓝色的线，就是这个例子里面，计算机得出来的最终的公式所画出来的图。

我们设定公式为 h(x) = θ1x + θ0 为计算机最终得出来的线性方程。

为什么假设的公式是 h(x) = θ1x + θ0 ？我也不知道 = = 或许这就是科学家厉害的地方，不知道怎么总结出来的……

（这里注意了，1和0是θ的下标。上面的公式你可以看成 h(x) = ax + b。之所以用θ，是因为图中用了θ）。

我们需要计算机计算出θ1和θ0两个未知参数。

既然h(x)为计算机的结果，那么与真是结果的差就是 h(x)-y 。那么每个差值加起来就是 (h(x0) - y0 ) + (h(x1) - y1 ) + … + (h(xn) - yn)。由于相减时会有负数的存在，为了不影响负数对加法的运算，我们将用每个差值的平方相加。这样并不会影响结论。

于是我们得出以下的公式。其中m为样本的数量。至于为什么有 1/2m，这或许是科学家和我的差距吧……

这个公式，就是我们所说的代价函数。我们要用这个公式去求θ1和θ0两个数的最优解。

这个公式要怎么解呢 ？ 这里我们先引入一个算法，叫梯度下降。

我们先假设 θ0 = 0 。那么 h(x) = θ1x 。这是一个很简单的一元函数。我们先用这个例子讲解以下梯度下降。

现在我们有三组数据 (1,1) (2,2) (3,3) 。

设如果 θ1 = 1 ，得出下面的图 ：

设如果 θ1 = 0.5 ，得出下面的图 ：

我们不断假设θ1的值，然后不断输入 x=1 , x=2 , x=3 后，通过代价函数，得出J(θ1)与输入值的关系图为 ：

我们上面说过，我们的目的就是要这个代价函数J(θ1)最小，求出参数θ1的数值。这个例子里面，当J(θ1)最小值时，θ1=1。1就是我们这个例子里面，最优解。

我们把θ0假设成0得出上面的结论。现在我们把θ0也考虑。不断假设θ1和θ0的数值，可以得出一个三维图 ：

这个三维图的最低点，就是代价函数J(θ0,θ1)的最小值。那么，在那个点对应的θ0、θ1值就是最优解。

如果用二维图像去表达上面的三位图，如下图：

每一个圈代表的J(θ)值相等，越靠近圈的中心，J(θ)值越小。所以就可以得到改点所对应的θ0与θ1。

那么这个不断假设值的过程我们是怎么进行的呢 ？
